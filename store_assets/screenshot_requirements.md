## Google Play Store Screenshots

### Required Sizes
- **Phone Portrait**: 1080 × 1920 pixels (16:9 aspect ratio)
- **7" Tablet Portrait**: 1200 × 1920 pixels
- **10" Tablet Portrait**: 1800 × 2560 pixels

### Screenshot Guidelines
- Use the same content sequence as iOS but optimized for Android Material Design
- Ensure screenshots show Android UI elements (navigation bar, status bar)
- Maintain consistent branding and messaging across platforms

## Design Guidelines

### Visual Style
- **Primary Colors**: Use Allma's brand color palette
- **Typography**: Clean, modern fonts with good readability
- **Spacing**: Generous white space for clean look
- **Shadows**: Subtle drop shadows for depth
- **Gradients**: Soft gradients for visual appeal

### Content Guidelines
- **Real Conversations**: Use authentic-looking conversation examples
- **Diverse Representation**: Show variety in companion personalities and appearances
- **Safety First**: Always include safety indicators where relevant
- **Privacy Focus**: Emphasize privacy and security features
- **Professional Quality**: High-resolution, pixel-perfect designs

### Text Overlays
- **Captions**: Clear, compelling headlines for each screenshot
- **Annotations**: Subtle callouts highlighting key features
- **Branding**: Consistent Allma branding throughout
- **Localization**: Prepare for multi-language versions

## Production Notes

### File Naming Convention
```
ios_iphone67_01_welcome.png
ios_iphone65_01_welcome.png
ios_ipad129_01_welcome.png
android_phone_01_welcome.png
android_tablet7_01_welcome.png
android_tablet10_01_welcome.png
```

### Quality Requirements
- **Resolution**: Native resolution for each device type
- **Color Space**: sRGB color profile
- **Format**: PNG with transparency support where needed
- **Compression**: Optimized for file size without quality loss
- **Accessibility**: High contrast ratios for text readability

### Localization Requirements
- Prepare screenshots in all supported languages
- Ensure text overlays are translatable
- Consider cultural sensitivities in conversation examples
- Maintain consistent visual design across languages

### A/B Testing Considerations
- Prepare alternative versions for key screenshots
- Test different messaging approaches
- Monitor conversion rates for screenshot variations
- Optimize based on user feedback and analytics

## Review and Approval Process

1. **Design Review**: Internal design team approval
2. **Marketing Review**: Marketing team messaging validation
3. **Legal Review**: Compliance and claims verification
4. **Localization Review**: Multi-language version validation
5. **Final Approval**: Executive sign-off before submission

## Update Schedule
- Screenshots should be updated with major app releases
- Seasonal updates for holiday themes (if applicable)
- Regular A/B testing of screenshot variations
- Immediate updates for any significant UI changes
